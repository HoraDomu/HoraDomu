Creator of Zen: A personal, lightweight, statically typed compiled language with clean syntax that produces native binaries as fast as C.
The philosophy: Python-style clarity in the syntax, C-style explicitness in the types.

Zen is not a scripting language. The syntax feels light and clean like one, but when you run a Zen program you are running a native executable compiled by your system C compiler, with no interpreter, no VM, and no garbage collector. Every variable has a type, every function declares its return type, and the compiler catches mistakes before anything runs. If it compiles, the types are correct.

Zen is not a systems language either there are no pointers, no manual memory management, no unsafe blocks. The runtime handles the low-level details while the compiler ensures the high-level ones.

That gap syntax that feels lightweight, output that is genuinely native is the niche Zen occupies. Most languages that feel this clean are interpreted. Most compiled languages feel heavier. Zen is neither.
